# 뷰 컴포넌트

  • 뷰 인스턴스

    • 생성자 함수
      new Vue({
    // ...
})
    • 인스턴스를 사용하는 이유
      • 생성자를 통해 생성한 객체에 여러 인스턴스를 활용하여 화면 단위 구성 가능

  • 뷰 컴포넌트

    • 컴포넌트를 사용하는 이유
      • 데이터의 흐름을 알 수 없는 기존의 MVC 패턴에서 탈피
      • 항상 상단에서 아래의 방향으로 흐르는 데이터의 흐름을 구성
      • 이를 위하여 컴포넌트 단위로 화면을 구성하고 데이터의 흐름을 구성
    • 컴포넌트 기본 문법
      Vue.component('my‐component', {
    // ...
});
    • 전역 컴포넌트
      Vue.component('app-content', {
    template:'<div>content</div>'
});
    • 지역 컴포넌트
      new Vue({
    el:'#app'
    ,data:{
        str:'App Header'
    }
    ,components:{
        'app-header':{
            template:'<h1>temp</h1>'
        }
    }
});

  • 뷰 컴포넌트 통신 방법

    • 컴포넌트 통신 규칙이 있는 이유?
      • 데이터의 흐름에 일관성을 갖게 하여 흐름 파악이 용이
      • 기존의 MVC 방식에서 탈피
    • props 속성
      • 모든 컴포넌트는 각각의 유효 범위를 가지고 있음
      • 서로 다른 컴포넌트의 데이터를 참조할 수 없음
      • 상단에서 하단으로 데이터의 흐름을 위하여 props 속성을 사용하여 데이터를 전달
      new Vue({
    el:'#app'
    ,data:{
        str:'App Header'
    }
    ,components:{
        'app-header':{
            props:['propsdata']
            ,template:'<h1>temp</h1>'
        }
    }
});
